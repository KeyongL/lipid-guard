# 📦 产品灵感与规格书 (v3.0 - 实用增强版)

## 💡 1. 核心目标
打造一个**真实可用**的本地饮食管理系统。重点在于**数据的可维护性**和**历史数据的可视化**。

## 🎯 2. 功能升级清单

### 🥗 A. 食物库管理 (Food Manager)
- **现状:** 只能从数据库读，没法加。
- **新需求:** - 新增 `/foods` 页面，展示所有食物列表。
  - 列表支持**搜索**（例如搜“肉”能列出所有肉类）。
  - 支持**新增食物**：表单输入名称、脂肪含量、风险等级，保存到数据库。
  - 支持**删除食物**：点垃圾桶图标删除。

### 📅 B. 记录管理 (Log Manager)
- **现状:** 记错了没法删，也不知道以前吃了啥。
- **新需求:**
  - 首页底部增加“今日记录明细”卡片。
  - 每条记录右侧显示“删除”按钮，点击后：
    1. 从数据库删除该条记录。
    2. **自动重新计算**今日仪表盘的进度条（回滚数据）。

### 📊 C. 视觉优化 (UI Polish)
- **卡片化:** 所有的功能区（仪表盘、记吃、明细）都要用圆角白色卡片 + 阴影区分，不要堆在一起。
- **色彩暗示:** - 绿灯食物显示清新的绿色标签。
  - 红灯食物显示警示的红色背景。

## 🛠 3. 技术约束
- **数据库:** 执行 `setup/seed_real_data.sql` 更新基础数据。
- **交互:** 删除操作必须有 `confirm` 弹窗确认，防止手滑。
